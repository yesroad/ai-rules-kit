---

## 진행 상황 추적

실행 시작 시 아래 항목을 TaskCreate로 등록한다. 각 Step 시작 시 `in_progress`, 완료 시 `completed`로 TaskUpdate한다.

- 계획 파일 읽기
- 구현
- 계획 대비 검증
- (FAIL 시) 재작업

---

## Step 1. 계획 파일 읽기

`.claude/plans/` 디렉토리에서 **가장 최근 수정된 `.md` 파일**을 읽는다.

```bash
ls -t .claude/plans/*.md 2>/dev/null | head -1
```

- 파일이 있으면 → 내용을 읽고 Step 2로 진행
- 파일이 없으면 → 사용자에게 안내:

> `.claude/plans/`에 계획 파일이 없습니다. `/start`로 먼저 계획을 생성하세요.

---

## Step 2. 계획 요약 출력

계획 파일에서 추출하여 아래 형식으로 출력한다:

```markdown
## 구현 시작: {작업 제목}

### 작업 내용
{계획의 작업 항목 목록}

### 변경 파일
{계획의 변경 파일 목록}

### 완료 조건
{계획의 완료 조건 체크리스트}
```

---

## Step 3. 복잡도 판단 및 구현 전략

> **복잡도 판단**: `@../workflows/thinking/model.md` 참조

| 복잡도 | 구현 방식 |
|--------|----------|
| LOW | 메인 컨텍스트에서 직접 구현 |
| MEDIUM | 메인 컨텍스트에서 직접 구현, 필요 시 explore 에이전트 |
| HIGH | general-purpose 에이전트에 위임 가능 |

---

## Step 4. 구현 진행

계획의 **작업 내용** 항목을 순서대로 구현한다.

**구현 원칙:**

- 계획의 변경 파일 목록을 따른다
- 기존 코드 패턴을 참조한다
- 계획 범위를 벗어나는 변경은 하지 않는다

**대규모 구현 시 에이전트 활용:**

```typescript
Task(
  (subagent_type = "general-purpose"),
  (prompt = `
    계획: {plan 파일 내용}
    구현 대상: {특정 작업 항목}
  `),
);
```

---

## Step 5. 계획 대비 검증

구현 완료 후 계획의 **완료 조건**을 하나씩 검증한다.

### 검증 항목

| # | 항목 | 검증 방법 | 결과 |
|---|------|----------|------|
| 1 | 작업 내용 체크리스트 | 각 항목별 구현 여부 확인 | PASS / FAIL |
| 2 | 변경 파일 목록 일치 | `git diff --name-only`와 계획 비교 | PASS / FAIL |
| 3 | `tsc --noEmit` | 타입 에러 없음 | PASS / FAIL |
| 4 | `lint` | 린트 오류 없음 | PASS / FAIL |
| 5 | 테스트 (계획에 포함 시) | 관련 테스트 실행 | PASS / FAIL / SKIP |

### 검증 결과 출력

```markdown
## 검증 결과

| 항목 | 결과 |
|------|------|
| 작업 내용 | ✅ PASS / ❌ FAIL ({실패 항목}) |
| 변경 파일 | ✅ PASS / ❌ FAIL ({누락/초과 파일}) |
| tsc | ✅ PASS / ��� FAIL ({에러 수}) |
| lint | ✅ PASS / ❌ FAIL ({에러 수}) |
| 테스트 | ✅ PASS / ❌ FAIL / ⏭️ SKIP |
```

---

## Step 6. FAIL 시 재작업 루프

하나라도 FAIL이면 자동으로 재작업한다:

1. FAIL 항목 분석
2. 수정 진행
3. Step 5 검증 재실행
4. 전체 PASS까지 반복 (최대 3회)

**3회 초과 시 사용자에게 보고:**

> 검증을 3회 시도했지만 아래 항목이 FAIL입니다:
> - {FAIL 항목 목록}
> 수동으로 확인이 필요합니다.

---

## Step 7. 완료 안내

전체 PASS 시 아래 안내를 출력한다:

```markdown
## ✅ 구현 완료

모든 완료 조건을 충족했습니다.

### 검증 결과
{Step 5 검증 결과}

### 다음 단계
`/done`으로 코드 검증 → 커밋 → PR을 진행하세요.
```
